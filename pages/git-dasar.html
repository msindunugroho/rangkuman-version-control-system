GIT

  1. Glosarry

    1. Repository

      Tempat penyimpanan untuk proyek perangkat lunak yang menggunakan Git untuk kontrol versi.

    2. Commit

      Tindakan menyimpan perubahan pada file atau proyek ke repositori Git.

    3. Branch

      Cabang dari repositori yang memungkinkan pengembang untuk bekerja secara terpisah dari kode yang ada.

    4. Merge

      Penggabungan perubahan dari satu branch ke branch lainnya.

    5. Pull Request

      Permintaan untuk menggabungkan perubahan dari satu branch ke branch lainnya, biasanya digunakan dalam kolaborasi tim.

    6. Push

      Mengirimkan perubahan lokal ke repositori Git yang terkait.

    7. Pull

      Mengambil perubahan dari repositori Git yang terkait dan menggabungkannya dengan perubahan lokal.

    8. Fetch

      Mengambil semua perubahan dari repositori Git yang terkait, tetapi tidak melakukan penggabungan otomatis dengan perubahan lokal.

    9. Clone

      Membuat salinan dari repositori Git yang ada di server ke komputer lokal.

    10. Remote

      Referensi ke repositori Git yang berada di tempat lain, biasanya digunakan untuk berinteraksi dengan repositori di server.

    11. Origin

      Nama default yang diberikan ke remote repository ketika melakukan clone.

    12. Revert

      Membatalkan perubahan tertentu pada commit tertentu.

    13. Fork

      Membuat salinan repositori milik orang lain di GitHub ke akun GitHub Anda sendiri.

  2. Konsep Dasar GIT

    1. Apa Itu GIT?

      Git adalah sebuah version control system yang sangat populer yang digunakan untuk mengelola perubahan dalam source code software. Diciptakan oleh Linus Torvalds pada tahun 2005, Git dirancang untuk menjadi ringan, cepat, dan memiliki dukungan untuk kolaborasi yang efisien.

    2. Instalasi GIT?

    3. Konfigurasi GIT (nama dan email)

      Perintah untuk mengatur konfigurasi nama dan email:

      git config --global user.name "input name here"
git config --global user.email "input email here"

      Untuk mengetahui nama dan alamat email git

      git config --global user.name
git config --global user.email

    4. Inisialisasi Repository

      Masuk ke repository/folder yang ingin di inisialisasi, lalu jalankan code ini:

      git init

      Jika repository telah terverifikasi terinisialisasi, maka git akan memberikan pesan verifikasi sebagai berikut:

      Initialized empty Git repository in /path/to/your/repository

    5. add dan commit file

      Konsep Staging Area dalam Git adalah tahap di antara direktori kerja Anda (working directory) dan repository lokal Anda. Ini adalah tempat di mana Anda menyiapkan perubahan sebelum Anda membuat commit permanen ke repository. Dalam tahap ini, Anda memilih file mana yang akan dimasukkan ke dalam commit berikutnya.

      Berikut adalah penjelasan langkah-langkah untuk menambahkan file ke staging area dan melakukan commit perubahan ke repository:

      • Tambahkan file/directory ke staging area

        // Menambahkan seluruh component(file, folder dsb) di dalam repository
git add .

// menambahkan component spesifik
git add "nama_folder/nama_file.txt"

      • Periksa status staging area

        Perintah ini akan menunjukan file-file yang telah di tambahkan dan atau yang belum di tambahkan ke staging area:

        git status

      • Lakukan Commit

        Standar Baku Pesan Commit 
        git commit -am "FIX: sloved carousel bug"
